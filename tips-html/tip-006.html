DEVTIP #006
Tokens as a Design Budget
Turn prompt length into an explicit cost model for features
Design Systems
WHY THIS MATTERS
Prompt size tends to grow until latency spikes, bills jump, and the agent starts missing key details. Treat tokens like a constrained palette: a budget that forces consistency, predictability, and reusable prompt parts.
The core idea: a token budget table
Define a budget by prompt layers and enforce it in code. A practical default for a feature-level agent call:
  • System instructions: 10–15%
  • Project / app context: 15–25%
  • Retrieved knowledge (RAG): 20–40%
  • User request: 5–10%
  • Output allowance: 20–30%
This turns “we ran out of context” into “retrieval exceeded its allocation” → a debuggable, reviewable event.
Example: encode budgets as config
// llm/budgets.ts
export type BudgetKey =
  | "system"
  | "appContext"
  | "retrieval"
  | "user"
  | "output";

export const DEFAULT_BUDGET: Record<BudgetKey, number> = {
  system: 700,
  appContext: 900,
  retrieval: 1600,
  user: 350,
  output: 900,
};

export const TOTAL_BUDGET = Object.values(DEFAULT_BUDGET).reduce(
  (a, b) => a + b,
  0
);
Example: trim each layer deterministically
// llm/trim.ts
import { encoding_for_model } from "tiktoken";

export function truncateToTokens(
  input: string,
  maxTokens: number,
  model: string = "gpt-4o-mini"
) {
  const enc = encoding_for_model(model);
  const tokens = enc.encode(input);

  if (tokens.length <= maxTokens) return input;

  // Keep the end for logs / recent context.
  const slice = tokens.slice(tokens.length - maxTokens);
  const out = enc.decode(slice);
  enc.free();
  return out;
}
Example: build prompts from “prompt components”
Co-locate prompt parts like UI components, then assemble with budgets.
// llm/promptParts.ts
export function systemPreamble() {
  return [
    "You are a senior TypeScript engineer.",
    "Return valid JSON that matches the schema.",
    "If unsure, ask for clarification in an 'issues' array.",
  ].join("\n");
}

export function brandVoice() {
  return [
    "Style: concise, direct, no fluff.",
    "Prefer bullet lists over paragraphs.",
  ].join("\n");
}
Fail closed when budgets are violated
If you only truncate, you may silently remove crucial facts. Add a budget report and refuse to run for high-stakes operations.
export function budgetReport(parts: Record<string, string>) {
  // If you can, compute real token counts; string length is a quick proxy.
  return Object.fromEntries(
    Object.entries(parts).map(([k, v]) => [k, v.length])
  );
}

export function assertNoHardTruncation(
  original: string,
  truncated: string,
  label: string
) {
  if (original !== truncated) {
    throw new Error(
      `Budget exceeded for ${label}. Refuse to run with missing context.`
    );
  }
}
PRO MOVE
Add token budgets to code review like design tokens. Make it a PR checklist item, then wire a CI check that prints the budget report for key agent flows. You catch prompt bloat before it hits production latency and cost.
