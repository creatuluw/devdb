ToolGate

The Idea

"Approval gates, schema contracts, and audit logs for AI agents using tools."

Market Developer Tools / AI Governance
Business Model B2B SaaS
Build Effort Medium — 4-week MVP
The Problem

AI agents can now call tools — send emails, create PRs, query databases, make purchases. But there is no standard way to enforce who approved what, what schemas are allowed, or to produce an audit trail that satisfies compliance teams.

"Browser agents are less safe than chat models — they need runtime guardrails that go beyond input/output filtering to cover actual tool execution."

— Invariant Labs, 2026

The Gap Today

Observability tools like LangSmith and Phoenix can trace what agents did after the fact. AWS Bedrock Guardrails filter inputs/outputs. But neither provides an app-agnostic policy gateway that sits between the agent and the tool — enforcing allow/deny/approval-required at runtime with typed schema contracts.

The Opportunity

MCP and structured tool schemas are making tool contracts a first-class surface. Enterprises adopting agents need exactly one thing before they go to production: a policy layer with human-in-the-loop approval and audit-grade logs. Nobody owns this layer yet.

The Solution

ToolGate
01

Policy Engine

Define allow/deny/approval-required rules per tool, per agent, per team. Write policies in YAML or via a visual editor. Supports wildcard matching, parameter constraints, and time windows.

02

Schema Contracts

Every tool call passes through a typed JSON Schema validator. Agents can only invoke tools with the exact parameters allowed. Malformed or unauthorized payloads are rejected before execution.

03

Human Approval Gates

High-risk actions (send email, delete record, make purchase) pause and route to a human via Slack, email, or dashboard. The agent waits for approval before proceeding — with configurable timeout and escalation.

04

Audit Trail

Every tool call — approved, denied, or timed out — is logged with agent ID, parameters, policy matched, approver, and timestamp. Exportable as JSON for compliance, queryable via API for other agents.

Why It's Valuable

shield

For Enterprises

Ship agents to production with confidence. Every tool call has a policy, an approval flow, and a log. SOC 2 and ISO 27001 auditors get the paper trail they need.

smart_toy

For Agents

Machine-readable policy responses. Agents know exactly which tools they can use, what parameters are allowed, and whether to wait for approval — no guessing.

engineering

For Developers

Drop-in middleware. Works with any MCP server, OpenAI function calling, or custom tool framework. One config file, zero code changes to your agent.

How to Build It

4-week MVP. Solo developer. Ships on Railway.

SvelteKit TypeScript Postgres Redis JSON Schema MCP SDK Railway Slack API
01

Policy Engine + Proxy

Build a lightweight HTTP proxy that intercepts tool calls. Policy rules in YAML define allow/deny/approval_required per tool name, agent ID, and parameter patterns. JSON Schema validation on every payload. Store policies in Postgres.

02

Approval Flow

When a call hits approval_required, ToolGate holds the request in Redis, sends a notification (Slack webhook or email), and exposes a /approve/:id endpoint. Configurable timeout + auto-deny/escalation.

03

Audit Log + Dashboard

Every decision (allow, deny, approve, timeout) is logged to Postgres with full context. SvelteKit dashboard shows real-time feed, filterable by agent, tool, and decision. Export as JSON or CSV for compliance.

04

SDK + MCP Integration

TypeScript SDK wraps any MCP client. Also ships as Express/Hono middleware. Agents query /capabilities to get their allowed tool list — machine-readable policy discovery. One-click deploy on Railway.
