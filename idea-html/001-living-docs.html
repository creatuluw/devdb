Daily Idea · April 10, 2026 · #001

LivingDocs

The Idea

"Auto-generated, always-fresh agent-readable knowledge layers for any website or product."

Market Developer Tools / AI Infra
Business Model B2B SaaS
Build Effort Medium — 4-week MVP
The Problem

AI agents are blind to most of the web. They hallucinate outdated API details or burn thousands of tokens parsing HTML noise. Only 5–15% of websites have implemented llms.txt, and most go stale within weeks.

"Without a dedicated machine-readable entry point, coding assistants fall back on generic web scraping, which frequently produces hallucinations or outdated patterns."

— Fern, 2026

The Gap Today

Google, AWS, and Microsoft have shipped MCP servers for their own docs. Every mid-size SaaS and open-source project has the same need — and zero budget to build this themselves.

The Opportunity

Gartner says 30%+ of new API demand in 2026 comes from LLM tools. ChatGPT traffic converts at 15.9%. Every website that wants to exist in an AI-first world needs this.

The Solution

LivingDocs
01

Paste any URL

Drop in a domain, sitemap, or GitHub repo. LivingDocs fetches the content, chunks it, and embeds it into a versioned knowledge base — automatically.

02

Generate llms.txt

An LLM pass produces a structured llms.txt and llms-full.txt, versioned and served at a permanent URL. Both humans and agents get a clean entry point.

03

Hosted MCP Server

A ready-to-use MCP endpoint exposes search_document and get_document tools. Any agent can plug in and query your content semantically.

04

Always Fresh

Webhooks for GitHub, Notion, and Gitbook trigger automatic re-indexing. A dashboard shows agent traffic, most-queried pages, and staleness indicators.

Why It's Valuable

person

For Humans

Your product shows up accurately in Cursor, Claude, and Copilot — not hallucinated. Higher AI-referred conversion traffic.

smart_toy

For Agents

Clean, token-efficient, semantic MCP entry point. No HTML parsing, no stale snapshots. Query any knowledge base in milliseconds.

trending_up

Compounding Moat

The longer a site is indexed, the richer the version history. Agents can ask "what changed in v2.3?" and get a real answer.

How to Build It

4-week MVP. Solo developer. Ships on Railway.

SvelteKit TypeScript Postgres + pgvector OpenAI Embeddings MCP SDK Railway Firecrawl GPT-4o-mini
01

Crawler + Extractor

SvelteKit frontend, URL input → sitemap parser → Firecrawl for content fetch → chunk + embed with text-embedding-3-small, store in Postgres with pgvector.

02

Generator

LLM pass (GPT-4o-mini) produces structured llms.txt and llms-full.txt. Version each generation. Expose at yourdomain.livingdocs.io/llms.txt.

03

MCP Server

Expose a minimal MCP-compatible endpoint: search_document + get_document tools over the embedded knowledge base. One-click deploy via Railway.

04

Sync + Dashboard

Webhook support for GitHub, Notion, Gitbook. Dashboard showing agent traffic (bot vs. human), most-queried pages, staleness indicators. Auto re-index on schedule.
