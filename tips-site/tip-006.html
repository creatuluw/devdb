DEVTIP #006

Tokens as a Design Budget

Turn prompt length into an explicit cost model for features

Design Systems
Today’s focus
schema
Budget prompts like UI tokens

Make token usage explicit, reviewable, and enforced.

WHY THIS MATTERS

Prompts expand quietly: more context, more retrieval, more instructions. Then latency jumps, costs spike, and the agent starts dropping key details. Treat tokens as a constrained palette — a design budget you enforce per layer — and your system becomes predictable, debuggable, and easier to evolve.

1) Define a budget table by prompt layer

Budgets force hard decisions. A practical baseline for a feature-level call: system (10–15%), app context (15–25%), retrieval (20–40%), user (5–10%), output (20–30%). Now “we ran out of context” becomes “retrieval exceeded its allocation” — a reviewable event. 

// llm/budgets.ts
export type BudgetKey =
  | "system"
  | "appContext"
  | "retrieval"
  | "user"
  | "output";

export const DEFAULT_BUDGET: Record<BudgetKey, number> = {
  system: 700,
  appContext: 900,
  retrieval: 1600,
  user: 350,
  output: 900,
};

export const TOTAL_BUDGET = Object.values(DEFAULT_BUDGET).reduce((a, b) => a + b, 0);

2) Trim deterministically (and choose what survives)

Truncation is inevitable. Make it deterministic and intentional: decide whether you keep the beginning (instructions) or the end (recent context/logs). The important part is that your behavior is stable across runs. 

// llm/trim.ts
import { encoding_for_model } from "tiktoken";

export function truncateToTokens(
  input: string,
  maxTokens: number,
  model: string = "gpt-4o-mini"
) {
  const enc = encoding_for_model(model);
  const tokens = enc.encode(input);

  if (tokens.length <= maxTokens) return input;

  // Keep the end for logs / recent context.
  const slice = tokens.slice(tokens.length - maxTokens);
  const out = enc.decode(slice);
  enc.free();
  return out;
}

3) Build a prompt “design system” of reusable parts

Treat prompt chunks like components: co-locate, version, and compose. The budget table becomes the interface contract between prompt design and runtime. 

// llm/promptParts.ts
export function systemPreamble() {
  return [
    "You are a senior TypeScript engineer.",
    "Return valid JSON that matches the schema.",
    "If unsure, ask for clarification in an 'issues' array.",
  ].join("\n");
}

export function brandVoice() {
  return [
    "Style: concise, direct, no fluff.",
    "Prefer bullet lists over paragraphs.",
  ].join("\n");
}

Then assemble with budgets: 

import { DEFAULT_BUDGET } from "./budgets";
import { truncateToTokens } from "./trim";
import { systemPreamble, brandVoice } from "./promptParts";

export function buildPrompt({ appContext, retrieved, user }: {
  appContext: string;
  retrieved: string;
  user: string;
}) {
  const system = truncateToTokens(
    systemPreamble() + "\n\n" + brandVoice(),
    DEFAULT_BUDGET.system
  );

  const ctx = truncateToTokens(appContext, DEFAULT_BUDGET.appContext);
  const rag = truncateToTokens(retrieved, DEFAULT_BUDGET.retrieval);
  const usr = truncateToTokens(user, DEFAULT_BUDGET.user);

  return { system, ctx, rag, usr };
}

4) Fail closed for high-stakes flows

Truncation can silently remove the one fact that makes the output safe. For billing, deletes, permissions, or production deploys, refuse to run when hard truncation occurs. 

export function budgetReport(parts: Record<string, string>) {
  // If you can, compute real token counts; string length is a quick proxy.
  return Object.fromEntries(
    Object.entries(parts).map(([k, v]) => [k, v.length])
  );
}

export function assertNoHardTruncation(
  original: string,
  truncated: string,
  label: string
) {
  if (original !== truncated) {
    throw new Error(
      `Budget exceeded for ${label}. Refuse to run with missing context.`
    );
  }
}
PRO MOVE

Make token budgets a code review artifact

Add a PR checklist item: “Did we change prompt parts and update the budget table?” Then run a CI check that prints a budget report for key agent flows. You will catch prompt bloat before it becomes production latency and cost.
