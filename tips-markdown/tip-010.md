# DevTip #010 -- Docs as a Build Artifact, Not a Write-Up

- Category: Documentation
- Date (UTC): 2026-04-14

## Subtitle
Make documentation reproducible: generated from code, gated in CI, and versioned with releases.

## Why this matters
Agent-built systems change fast, and manual docs decay even faster. The fix is to treat documentation like a build artifact: generated from source, validated in CI, and published from the same pipeline that ships the code. When the agent that writes the code also emits the docs, accuracy is &ldquo;by construction&rdquo; and drift becomes a failing test, not a surprise.

## Tip
### 1) Define a single doc contract: types first
Start with stable types and invariants, then generate outward (OpenAPI, SDK stubs, and human docs). The key is a single source of truth that tools can read deterministically.

```ts
// packages/api/src/contracts/user.ts

export type UserId = string & { readonly __brand: 'UserId' };

export interface User {
  id: UserId;
  email: string;
  createdAt: string; // ISO-8601
}

/**
 * Invariant: email must be normalized and lowercase.
 */
export function normalizeEmail(email: string) {
  return email.trim().toLowerCase();
}

```

### 2) Generate OpenAPI from code (and fail fast on drift)
If you already have TS types, generate OpenAPI during build, then validate it. In CI, compare against the committed spec so drift is explicit.

```ts
// scripts/gen-openapi.ts
import { writeFileSync } from 'node:fs';
import { generateOpenApi } from './openapi-generator';

const spec = generateOpenApi();
writeFileSync('openapi.json', JSON.stringify(spec, null, 2));

// Then in CI: diff openapi.json vs repo version

```

### 3) Turn OpenAPI into living docs (SvelteKit page from spec)
Treat the spec as data. A SvelteKit route can render endpoint tables, request/response examples, and error codes from the current build output.

```ts
// src/routes/docs/api/+page.server.ts
import spec from '../../../../openapi.json';

export const load = async () => {
  const paths = Object.entries(spec.paths).map(([path, methods]) => ({
    path,
    methods: Object.keys(methods)
  }));

  return {
    title: spec.info.title,
    version: spec.info.version,
    paths
  };
};

```

### 4) Add a documentation quality gate in CI
Do not just build docs; verify them. Lint generated Markdown, check for broken internal links, and ensure the published docs version matches the package version.

```ts
# .github/workflows/docs.yml

- name: Generate docs
  run: pnpm docs:build

- name: Check broken links
  run: pnpm docs:check-links

- name: Enforce version sync
  run: node scripts/check-doc-version.mjs

```

## Pro Move
Have the agent emit an ADR (Architecture Decision Record) alongside each non-trivial change. Store ADRs in the repo and generate a &ldquo;Decisions&rdquo; index page at build time. This turns tribal knowledge into searchable, versioned context that future agents can load as ground truth.
